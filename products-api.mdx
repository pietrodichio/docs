## Overview

The Products API enables you to manage your product catalog through three main operations: bulk import, bulk update, and bulk delete. These endpoints are protected and require authentication using a JWT token.

## Authentication

All endpoints require a valid JWT token to be included in the Authorization header:

```bash
Authorization: Bearer your_jwt_token
```

### Bulk Update Products

<ParamField path="bulk-update" type="POST" required>
  Update multiple products at once in your catalog
</ParamField>

#### Body Parameters

<ParamField
  body="products"
  type="array"
  required
  initialValue={[
  {
    id: "prod_12345",
    title: "Updated Product",
    description: "An updated product description",
    image: "https://example.com/updated-image.jpg",
    url: "https://yourstore.com/products/updated-product",
    variants: [
      {
        id: "var_12345",
        title: "Updated Variant",
        price: "39.99",
        inventory_quantity: 75,
        variant_url: "https://yourstore.com/products/updated-product/variant",
      },
    ],
    collections: ["Best Sellers"],
  },
]}
>
  Array of products to update. The structure is identical to the bulk import
  endpoint. Only include the fields you want to update - omitted fields will
  remain unchanged.
</ParamField>

### Bulk Delete Products

<ParamField path="bulk-delete" type="DELETE" required>
  Delete multiple products at once from your catalog
</ParamField>

#### Query Parameters

<ParamField query="ids" type="string" required initialValue="prod_12345,prod_67890">
  Comma-separated list of product IDs to delete
</ParamField>

## Rate Limiting

The API implements rate limiting to ensure fair usage. The current limits are:

* Bulk Import: 1000 products per request

* Bulk Update: 500 products per request

* Bulk Delete: 200 products per request

## Best Practices

1. **Batching**: When dealing with large numbers of products, break them into smaller batches to avoid timeouts and rate limits.

2. **Retries**: Implement exponential backoff for retries when encountering rate limits or temporary failures.

3. **Validation**: Always validate your product data before sending it to avoid partial updates.

4. **Idempotency**: Operations are idempotent - duplicate requests with the same data won't create duplicate products.
